## 目的とKPI

- 目的: **Starter（月¥50,000）の獲得**。X → プロフィール → LP（/starter）→ Stripe決済。
- 見る指標: プロフィールクリック → LP流入 → Stripe決済。
- 見ない指標: フォロワー数単体・インプ単体。

## 週次サイクル（誰が・いつ・何を）

| 曜日 | やること | 担当 |
|---|---|---|
| 月曜 | 先週の投稿データ分析（実数ベース） | **AI**（X APIで取得可能。分析結果は人が確認） |
| 火曜 | 来週分14投稿のドラフト生成 + 機械チェック | **AI**（スキル `x-post-weekly-generation`） |
| 水曜 | ドラフトの選別・リライト・`【実数】`プレースホルダの充填 → 本番シートへ | **人**（数字を入れられるのは人だけ） |
| 毎日 | 予約時刻に自動投稿（GitHub Actions） | **自動**（監視ログ: `logs/post_summary.md`） |
| 毎日 | リプライ10〜20件（下記「リプライの質」参照） | **人**（AIは下書き支援のみ） |
| 週1 | 「今週の個人開発リリース5選」 | **人**がネタ収集 → **AI**が講評ドラフト |

## 自動化の線引き

**AIに任せてよいもの:**
- 投稿ドラフトの生成・重複チェック・文字数/NGワード検査（`scripts/check_posts.py`）
- 投稿の実行（パイプライン稼働中。触るときは必ず `DRY_RUN=1`）
- 投稿メトリクスの取得と月次分析（X API Bearer Tokenで自アカウントの実数を取得できる）
- リプライの「下書き」: 相手のプロダクトURLをもらえば、LP・導線の具体的フィードバック案を生成

**人にしかできないもの:**
- 実数の確定（AIの数字はすべて `【実数】` プレースホルダ。捏造禁止）
- 投稿の最終選別と予約（水曜）
- リプライの送信・DM・関係構築
- X記事（Articles）の執筆・公開判断

## 投稿ルールの要点（詳細は x_strategy_workle.md）

1. 勝負は1行目・冒頭3行。数字・固有名詞・断定を1行目に入れる
2. 外部リンクは本文に貼らない（リプ欄にUTM付き。X内の記事リンクは例外でOK）
3. 実数のみ使用。出せる数字がない間は「数字を出す」のではなく「過程を見せる」
4. 情報商材トーン禁止（「稼げる」「限定」系）

## 実データから見えた勝ちパターン（2026-07-14時点・フォロワー112人）

直近3週間の実測（インプレッション）:

- **X記事（Articles）へのリンク投稿: 413imp** — 週間最高。記事はXが配信を優遇している気配があり、継続の価値が高い
- **共感フック型「◯◯な個人開発者へ」: 106〜281imp** — 最も安定して伸びる型
- **対比構文型「伸びる開発者は◯◯／伸びない開発者は△△」: 78〜299imp**
- **Workle自己紹介（固定ポスト系文面）: 272〜275imp・いいね12〜17** — 刺さっている
- 死んだ型: 単発のリプ誘導質問（20〜49imp）、数字のないBuild in Public実況（20〜40imp）

### リプライの質（最重要の改善点）

量は規定通り（10〜20件/日）だが、内容がほぼ一言の相槌・賞賛になっている。
戦略の規定は**「売り込みゼロの、LP・導線・訴求への具体的フィードバック」**。
相槌はタイムラインの露出にはなるがプロフィールクリックに繋がらない。
リプライ単体で715impを取った例もあり、リプ欄の露出は本投稿より大きい。
「このLPのファーストビュー、◯◯を△△にすると伝わる」型に切り替えると、リプライがそのまま営業資産になる。

## 数字ゼロ期（現在）の運用方針

フォーマルな検証前で出せる実数がない間:

- 数字の代わりに**「過程の一次情報」**を出す（今日何を作ったか、何を試して何が空振りだったか）
- 検証が終わって数字が出たら、その実数が最初の「実数開示型」投稿・note・LP掲載実績になる
- LP（/starter）も同方針: 事例・数字は載せず「盛らずに過程ごと公開する」ポジションで統一済み

## 今後AIが自動で回せるもの（依頼すれば即）

1. 毎週月曜の**実数分析レポート**（X APIから全投稿メトリクス取得 → 勝ち/負け共通点の抽出 → このファイルに追記）
2. 毎週火曜の**14投稿ドラフト生成**（定期実行のスケジュール化も可能）
3. リリース報告への**フィードバックリプ下書き**（URLを渡すだけ）
4. X記事の**ドラフト執筆**（テーマと骨子をもらえれば本文まで）
